# Theming API - Basics

## Design a button

Let's take this further and try to create a custom button component. The button
will have 2 sizes, `small` and `large`, and 2 variants `outline` and `solid`.

<ButtonSpec />

First, let's define this spec in the `register` object. The button has just a
container, for now, so we're use register only this part.

```jsx live=false
const register = {
  parts: ["container"],
  sizes: ["small", "large"],
  variants: ["outline", "solid"],
}
```

> The inevitable fact is that the designer would made this, will most likely add
> more component parts, like spinners, and icons later on. This API makes it
> easy for us to update.

<br />

Next, we define the `baseStyle` for each part, adding common attributes like
`font-weight`, `text-transform`, etc.

```jsx live=false
const baseStyle = {
  container: {
    fontWeight: "semibold",
    borderRadius: "4px",
    textTransform: "uppercase",
  },
}
```

Next, we'll define the styles for the button sizes

```jsx live=false
const sizes = {
  small: {
    container: {
      paddingX: "18px",
      paddingY: "12px",
      fontSize: "14px",
    },
  },
  large: {
    container: {
      paddingX: "24px",
      paddingY: "16px",
      fontSize: "18px",
    },
  },
}
```

Now, let's add 2 variant styles to the button, `solid` and `outline`. Here's how
to do it:

```jsx live=false
const variants = {
  solid: {
    container: {
      bg: "#4053B8",
      color: "white",
    },
  },
  outline: {
    container: {
      border: "2px",
      borderColor: "#4053B8",
      color: "#4053B8",
    },
  },
}
```

Bringing it all together, We now have a themable Button component. Here's what
it looks like:

```jsx manual=true
const styleConfig = {
  register: {
    parts: ["container"],
    sizes: ["small", "large"],
    variants: ["outline", "solid"],
  },
  baseStyle: {
    container: {
      fontWeight: "semibold",
      borderRadius: "4px",
      textTransform: "uppercase",
    },
  },
  sizes: {
    small: {
      container: {
        paddingX: "18px",
        paddingY: "12px",
        fontSize: "14px",
      },
    },
    large: {
      container: {
        paddingX: "24px",
        paddingY: "16px",
        fontSize: "18px",
      },
    },
  },
  variants: {
    solid: {
      container: {
        bg: "#4053B8",
        color: "white",
      },
    },
    outline: {
      container: {
        border: "2px",
        borderColor: "#4053B8",
        color: "#4053B8",
      },
    },
  },
}

function Button(props) {
  const { size, variant, ...rest } = props

  const styles = useStyleConfig("Button", {
    size,
    variant,
    styleConfig,
  })

  return <Box as="button" sx={styles.container} {...rest} />
}

render(
  <Button size="small" variant="solid">
    Submit
  </Button>,
)
```

### Setting default values

Most of the time, you want the button component to come with a default `size`
and `variant` without having to pass props to it.

To leverage React's `defaultProps` to handle this, or pass a `defaultProps`
object to the style config.

Let's say we want to set the button size to `small` and variant to `solid` by
default. Here's what we'll do:

```jsx live=false
Button.defaultProps = {
  size: "small",
  variant: "solid",
}
```

That's it! We've created a button component style that makes it easier to manage
current and future changes consistently.

## Design a badge

One more example!

Let's say you need to create a simple badge component with the following spec.

<BadgeSpec />

First thing we notice is that the badge has **base or common styles** across all
its variants. Here's how we'll model that:

Define the spec in the `register`

```jsx live=false
const register = {
  parts: ["container"],
  variants: ["in-progress", "removed", "new", "default"],
}
```

Define the base style

```jsx live=false
const baseStyle = {
  container: {
    fontSize: "12px",
    fontWeight: "bold",
    textTransform: "uppercase",
    color: "white",
  },
}
```

Define the variants

```jsx live=false
const variants = {
  "in-progress": {
    container: { bg: "blue.600" },
  },
  new: {
    container: { bg: "purple.600" },
  },
  removed: {
    container: { bg: "red.600" },
  },
  default: {
    container: { bg: "gray.600" },
  },
}
```

Bringing it all together:

```jsx manual=true
const styleConfig = {
  register: {
    parts: ["container"],
    variants: ["in-progress", "removed", "new", "default"],
  },
  baseStyle: {
    container: {
      fontSize: "12px",
      fontWeight: "bold",
      textTransform: "uppercase",
      color: "white",
      px: "2",
      py: "1",
      borderRadius: "4px",
    },
  },
  variants: {
    "in-progress": {
      container: { bg: "blue.600" },
    },
    new: {
      container: { bg: "purple.600" },
    },
    removed: {
      container: { bg: "red.600" },
    },
    default: {
      container: { bg: "gray.600" },
    },
  },
}

function Badge(props) {
  const { size, variant, ...rest } = props
  const styles = useStyleConfig("Badge", { size, variant, styleConfig })
  return <Box as="span" sx={styles.container} {...rest} />
}

// Pro tip: Set the default variant with `defaultProps`
Badge.defaultProps = {
  variant: "default",
}

render(<Badge>Chakra</Badge>)
```

That's it! Now you have a solid structure your team can use to create new
variants or improve the base styles.

And of course you could always move these styles into the central theme if you'd
prefer to maintain it there.

# Icon

The icon component renders `svg` icons. Chakra UI provides basic interface
icons. To add custom icons, [read the guide](#adding-custom-icons).

> Avoid passing `onClick` handlers to the `Icon` component. If you need a
> clickable icon, use [IconButton](/components/icon-button) instead.

## Import

```js
import { Icon } from "@chakra-ui/core"
```

## Usage

Use an icon by passing the `name` prop. This name must match an icon key in
`theme.icons`. By default, the icon inherits the `fontSize` and `color` of its
parent.

```jsx
<Stack direction="row">
  {/* Default size is 1em => 16px */}
  <PhoneIcon />

  {/* Use the `size` prop to change the icon size */}
  <CheckCircleIcon size="24px" />

  {/* Use the `color` prop to change the icon color */}
  <WarningIcon boxSize="32px" color="red.500" />
</Stack>
```

## Using an icon library

You might need to use icons from a popular icon library like
[react-icons](https://react-icons.netlify.com/). Here's how to go about it:

```jsx live=false
// 1. import the icon from "react-icons"
import { MdSettings } from "react-icons/md"

// 2. Use the `as` prop to render the icon
<Icon as={MdSettings} />

// 3. Then you can pass all `Box` props to style the icon
<Icon as={MdSettings} size="32px" color="green.400" />
```

Here's a working example:

```jsx

```

## Adding custom icons

You can also create your own custom icons using the `createIcon` utility:

```jsx live=false
import { createIcon } from "@chakra-ui/icon"
export const UpDownIcon = createIcon({
  displayName: "UpDownIcon",
  viewBox: "-1 -1 9 11",
  d:
    "M 3.5 0L 3.98809 -0.569442L 3.5 -0.987808L 3.01191 -0.569442L 3.5 0ZM 3.5 9L 3.01191 9.56944L 3.5 9.98781L 3.98809 9.56944L 3.5 9ZM 0.488094 3.56944L 3.98809 0.569442L 3.01191 -0.569442L -0.488094 2.43056L 0.488094 3.56944ZM 3.01191 0.569442L 6.51191 3.56944L 7.48809 2.43056L 3.98809 -0.569442L 3.01191 0.569442ZM -0.488094 6.56944L 3.01191 9.56944L 3.98809 8.43056L 0.488094 5.43056L -0.488094 6.56944ZM 3.98809 9.56944L 7.48809 6.56944L 6.51191 5.43056L 3.01191 8.43056L 3.98809 9.56944Z",
})
```

The major advantage of `createIcon` is that it forwards ref properly.

If you use <Icon as={MdSettings}/> and `MdSettings` doesn't use `forwardRef`, the ref doesn't get forwarded automatically. This may report a React error in the console about passing ref to a function component. In short, you can use `<Icon as={MdSettings} />` if you know that `MdSettings` is forwarding the ref - otherwise, use the `createIcon` utility function and then call the icon directly: `<MdSettings />`.

Before generating your custom icon do the following:

- Export the icon's `svg` from Figma, Sketch, etc.
- Use a tool like [SvgOmg](https://jakearchibald.github.io/svgomg/) to reduce
  the size and minify the markup.
- Use `generateIcon` to create an icon as shown above.

Start using your icon right away. There is no need to update your theme files.

## Props

Icon component composes `Box` so you can pass all `Box` props.

| Name        | Type                  | Default        | Description                                                                         |
| ----------- | --------------------- | -------------- | ----------------------------------------------------------------------------------- |
| `size`      | `string`              | `1em`          | The size of the icon.                                                               |
| `name`      | `string`              |                | The name of the icon.                                                               |
| `color`     | `string`              | `currentColor` | The color of the icon.                                                              |
| `focusable` | `boolean`             | `false`        | Denotes that the icon is not an interative element, and only used for presentation. |
| `role`      | `presentation`, `img` | `presentation` | The html role of the icon.                                                          |
